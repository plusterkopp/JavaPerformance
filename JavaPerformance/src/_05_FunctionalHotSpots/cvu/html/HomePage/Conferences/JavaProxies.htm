ABSTRACT
 
Java 1.3, among other new features, introduces two novel abstractions: dynamic proxies and the invocation handler. These two features open the way for programming techniques that are new to Java, but well-known from other programming languages.

With dynamic proxies and the invocation handler we can define new classes and modify the behavior of existing objects at runtime, a concept that makes for a great deal of flexibility in our programs.

Examples of use of dynamic proxies include:

  • generic pre- and post-processing for all methods of a class; this is similar to techniques that we know from aspect-oriented programming languages
  • adding functionality to classes at runtime; this is similar to mix-in techniques used in languages that have multiple inheritance (in a way it makes up for the lack of multiple inheritance in Java)
Naturally, these are just sample usages, but they might give a feeling for dynamic proxies and how powerful a concept they add to Java in practice, although its no more than just two new features in JDK 1.3.

Along with the case studies we'll give an evaluation of the pros and cons of programming with dynamic proxies in practice.
