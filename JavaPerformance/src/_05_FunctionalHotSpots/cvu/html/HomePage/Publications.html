Here is a list of publications that I wrote together with Klaus Kreft. The list includes articles published as regular columns, for instance in C++ Report, C/C++ Users Journal, and JavaSPEKTRUM, as well as a number of feature article that were published in various magazines.  The list of articles is ordered by topic or by magazine.  Go to JAVA for Java related papers and to C++ for C++ related papers.  A brief description of the magazines that we published in can be found below.

In addition to material published in printed and online magazines, I've been putting together a FAQ document about Java Generics.  It is available on this website at FAQ.

Furthermore, we've been writing numerous book reviews, most of which were published in print or online magazines. To browse through the book reviews go to REVIEWS. The choice of books under review is somewhat arbitrary and is not meant as a list of recommended reading.  For book recommendations go to RECOMMENDED.
 

     


    since 2002

    		   Java SPEKTRUM 
    (SIGS DATACOM / 101 Communications)
    COLUMN:    EFFECTIVE JAVA

    JavaSPEKTRUM is a German magazine for Java programmers.  All articles of this column are available in German only. 

    The column is devoted to the Java programming language and its platform libraries.  It covers introductory to in-depth Java topics, some articles being purely introductory, the majority pointing out pitfalls in the language and explaining best practice programming techniques.

    1997-2000
    		 C++ Report 
    (SIGS Publications)
    COLUMN:   EFFECTIVE STANDARD C++ LIBRARY

    C++ Report used to be a high caliber C++ magazine published in the USA. Effective June 2000 the  publisher discontinued publication of C++ Report. Our column "Effective Standard Library" was continued in C/C++ Users Journal (CUJ).

    The "Effective Standard Library" column explained various aspects of the standard library, including iostreams, locales, autopointers, and of course the STL with its iterators, functor, containers, and algorithms.
